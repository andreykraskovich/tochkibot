# tochkibot — Прогнозист ЧМ 2026

Telegram-бот для прогнозов на матчи ЧМ 2026. Матчи автоматически подтягиваются с football-data.org.

## Команды

| Команда | Описание |
|---|---|
| `/today` | Матчи сегодня |
| `/upcoming` | Ближайшие матчи |
| `/predict` | Сделать прогноз (кнопки) |
| `/leaderboard` | Таблица прогнозистов |
| `/results` | Результаты завершённых матчей |
| `/sync` | Принудительная синхронизация (только админ) |
| `/myid` | Узнать свой Telegram ID |

## Переменные окружения

```
BOT_TOKEN=токен от BotFather
DATABASE_URL=postgresql://...
FOOTBALL_API_KEY=ключ от football-data.org
ADMIN_IDS=123456789,987654321
CHAT_ID=-100xxxxxxxxxx
WC_COMPETITION_ID=2000
```

- `ADMIN_IDS` — Telegram ID тех, кто может делать /sync (через запятую). Узнать через /myid.
- `CHAT_ID` — ID группового чата для ежедневных уведомлений в 9:00 МСК. Узнать: добавить бота в чат, написать /myid — покажет ID чата тоже. Или использовать @userinfobot.
- `WC_COMPETITION_ID` — ID турнира на football-data.org. По умолчанию 2000 (ЧМ). Проверить через /sync и логи.

## Как получить CHAT_ID

1. Добавь бота в группу
2. Напиши в группе любое сообщение
3. Открой: `https://api.telegram.org/bot<BOT_TOKEN>/getUpdates`
4. Найди `"chat":{"id":...}` — это и есть CHAT_ID (отрицательное число для групп)

## Деплой на Railway

1. Подключить репозиторий в Railway
2. Добавить все переменные окружения
3. Подключить существующую PostgreSQL базу

## Очки

- 3 очка за правильно угаданный исход (победа хозяев / ничья / победа гостей)
- 0 очков за неправильный прогноз
- Прогноз можно изменить до начала матча

### Плей-офф

В матчах плей-офф исход считается по **основному времени** (90 минут):

- **П1 / П2** — если победитель определился в основное время.
- **Ничья** — если в основное время была ничья, даже если потом матч ушёл в
  дополнительное время или серию пенальти и кто-то победил.

Технически: API football-data.org отдаёт `score.regularTime` — счёт ровно за
90 минут. Он есть только когда матч вышел за пределы основного времени (плей-офф),
поэтому счёт матча мы берём как `regularTime ?? fullTime`. Важно: у пенальтийных
матчей `fullTime` содержит счёт серии пенальти (например, 3–0), а не игровой счёт —
именно поэтому считать нужно по `regularTime`. Метки `(д.в.)` / `(пен.)` в выводе
показывают, что в основное время была ничья, а дальше были доп.время / пенальти.
